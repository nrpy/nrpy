Defusing the fused_rkf45 kernel:
"""
Singularity-Axiom: Modular Physics Pipeline

MISSION:
Decompose the fused RKF45 integration into a "Single-Instruction Multiple-Data" (SIMD) batch pipeline. This prevents register pressure from exceeding the 255-register limit of the RTX 3080 (sm_86) during complex numerical spacetime evaluations.

ARCHITECTURAL FLOW:
The orchestrator must launch the following three kernels sequentially for each of the 6 RKF45 stages:

    Batch Interpolation Kernel (placeholder_interpolation):
    Input: Current state bundle $f^\mu$ (VRAM).
    Output: Symmetric metric $g_{\mu\nu}$ and Christoffel symbols $\Gamma^\alpha_{\beta\gamma}$ stored in VRAM-resident bundle arrays.
    Optimization: Utilize BHAH_HD_FUNC for zero-overhead tensor evaluation.
    Goal: Minimize VRAM stalls by ensuring coalesced memory access across the bundle.

    Batch RHS Kernel (calculate_ode_rhs):
    Input: State bundle $f^\mu$ + Interpolated tensor bundle (VRAM).
    Output: Derivative bundle $\dot{f}$ (the $k$-array).
    Constraint: Output MUST be written to Global VRAM. Do NOT use Shared Memory, as this data must persist for the subsequent Update kernel.
    Logic: Solve the Geodesic Equation: $\frac{d^2x^\mu}{d\lambda^2} + \Gamma^\mu_{\alpha\beta}\frac{dx^\alpha}{d\lambda}\frac{dx^\beta}{d\lambda} = 0$.

    Adaptive Stage-Update Kernel (rkf45_stage_update):
    Input: int stage, current $f_{start}$, $\dot{f}$ bundle, and step size $h$.
    Logic: Use a switch(stage) block to apply specific RKF45 coefficients (e.g., Stage 3 uses $3/32 \cdot k_1 + 9/32 \cdot k_2$).
    Memory: Update the f_temp bundle in VRAM for the next interpolation cycle.

CONSTRAINTS:

Persistence: 
All intermediate data (metrics, connections, and RHS derivatives) must be stored in strided Structure of Arrays (SoA) bundles to protect the 10GB VRAM limit.

No Synchronization: 
Do not synchronize the GPU between the Interpolation, RHS, and Update kernels unless the hardware requires it; allow the Compute Command Processor to schedule them as a continuous stream.

Metadata: 
All kernels must access global tolerances and physics constants via Constant Memory (d_commondata).

    * Input: d_f_bundle, d_metric_bundle.

        2. Solve the Hamiltonian Constraint $g_{\mu\nu}p^\mu p^\nu = 0$ for the negative root of $p^t$.

    * Input: d_f_temp_bundle, d_metric_bundle, d_connection_bundle.
    * Output: d_k_bundle (The specific $k_{stage}$ vector).

        1. Load geometry tensors from VRAM (bypassing re-calculation).
        2. Compute the geodesic derivatives $\dot{f} = \{ \frac{dx^\mu}{d\lambda}, \frac{dp^\mu}{d\lambda} \}$.

    * Generator Script:  rkf45_stage_update.py
    * Host Launcher Function: rkf45_stage_update(const double *restrict d_f_start, const double *restrict d_k_bundle, const double *restrict d_h, const int stage, const int chunk_size, double *restrict d_f_temp)

        - Threads: 256 per block.
        - Blocks: (chunk_size + 255) / 256.
    * Input Buffers (VRAM):
        - d_f_start: Base state vector at t_n [Size: 9 * chunk_size].
        - d_k_bundle: Flattened derivative array [Size: 6 * 9 * chunk_size].
        - d_h: Step size array [Size: chunk_size].

        4. Computation: Uses `cuda_intrinsics.h` (FusedMulAddCUDA) to compute $f_{temp} = f_n + h \sum (a_{ij} k_j)$.

        - Uses strictly flattened 1D arrays with SoA layout (Component-Major within the bundle).
        - Macros `IDX_F(c, ray)` and `IDX_K(s, c, ray)` are defined locally to ensure hermetic compilation.

    * Host Launcher Function: rkf45_finalize_and_control(double *restrict d_f_persistent, const double *restrict d_f_start, const double *restrict d_k_bundle, double *restrict d_h, termination_type_t *restrict d_status, double *restrict d_affine, int *restrict d_retries, const long int chunk_size)
    * Grid Configuration:
        - Threads: 256 per block.
        - Blocks: (chunk_size + 255) / 256.
    * Input Buffers (VRAM):
        - d_f_start: Base state vector [Size: 9 * chunk_size].
        - d_k_bundle: All 6 calculated derivative vectors [Size: 54 * chunk_size].
        - chunk_size: Number of active rays.

        - d_affine: Affine parameter $\lambda$.
        - d_retries: Retry counter for the current step.

        3. Error Estimation:

             * Increments `d_retries`.
             * Shrinks `d_h` by 0.5 (limited by `rkf45_h_min`).
             * Sets status to REJECTED (or FAILURE if retries > limit).

        2. If plane crossing detected: Root-find intersection $t_{event}$, Interpolate state.
        3. Update Blueprint buffer and mark TERMINATED if applicable.

     * `d_metric_bundle`: Storage for $g_{\mu\nu}$.

       - Input: `d_f_temp`, `d_metric`, `d_connection`.

       - Call `rkf45_stage_update_kernel`.
       - Input: `d_f_bundle` (Base), `d_k_bundle`, `d_h_bundle`, `stage`.
